HTML 작성 규칙

1. 기본

  • 파일 링크 경로 ‘상대경로’로 작업
    - 모바일웹, 모바일앱(하이브리드앱) HTML 동일 사용 : 개발작업 웹/앱 별도
    - 모바일앱 페이지 기준으로 상대경로 작업
      CSS: ../../../resource/css/mwp/mweb.css
      JS: ../../../resource/js/mwp/
      Image: ../../../resource/img/mwp/
  • 동일한 이름의 id는 문서에서 한번만 사용 가능
  • 그룹간 객체 구분을 위한 빈줄 허용하지 않음(필요한 경우 1줄을 초과하지 않음)
  • 종료 태그 필요 없는 엘리먼트:<br>,<col>,<hr>,<img>,<input>,<link>,<meta>...
  • 특정 엘리먼트에 id, class, style의 선언 순서는 기본 애트리뷰트 지정 후 선언
  • 콘텐츠 영역의 주석 표기는 선택 사항

2. 구조와 표현의 분리

  • 테이블 셀의 경우 다양한 너비의 고정폭을 지닐 수 있으므로 예외의 경우를 허용
  • input[text] 엘리먼트의 경우 너비의 고정폭 지정 허용함

3. 엘리먼튼 필수 요소

엘리먼트 종류 속성 기본 값 비고
input 입력 type text password, email, number, tel, search, date
input 선택 type radio
checkbox
name 빈 값(혹은 임시 값)
id label 이 지정된 경우 label 엘리먼트의 for 값과 동일한 id 를 명시
title label 엘리먼트가 지정되지 않은 경우 title 작성
label for radio,checkbox의 id값과 동일한 이름으로 연결하며 명시적 사용을 권장
button type button
textarea cols 기본값 10 지정
rows 기본값 3 지정
select 디자인 적용 별도 마크업(Layer)

4. 명명규칙

  • 파일 ID 기준 작성 : SB, PSD 파일명 동일

5. 구조(Layout) id

  • #wrap : 페이지 전체 영역 그룹
  • #header : 최상단영역 그룹
  • #contents : 본문영역 그룹
    .ctg_폴더명 : 카테고리(폴더)별 컨텐츠영역 그룹
  • #footer : 하단영역 그룹

6. HTML 작업 확인 범위

  • 작업이 완료된 HTML 파일 확인
    - 크롬 웹 브라우저
    - 안드로이드 기본 브라우저
    - 아이폰 기본 브라우저
    - 각 디바이스 크롬 설치 확인
  • 타 앱뷰의 기능 버튼 제어 불가(모바일웹)

7. 주석

  • 개발전달 이후 수정사항 적용 시 수정된 날짜 및 내용으로 표기해 준다.
  • 작성규칙 : <!-- 수정일 수정(or 추가) : 수정내용 >
  • 한줄이 수정된 경우 :
    <h2 class="cnt_tit">타이틀</h2> <!-- 2018-07-24 수정 : 텍스트 -->
  • 두줄이상 수정된 경우 :
    <!-- 2018-07-24 수정 : 텍스트 --> <div class="cnt_wrap"> <h2 class="cnt_tit">페이지타이틀</h2> </div> <!-- //2018-07-24 수정 : 텍스트 -->
