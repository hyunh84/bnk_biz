CSS 작성 규칙

1. 기본

  • 한개의 파일에서 CSS 파일을 @import로 추가 호출 한다.
  • 모든 속성은 영문 소문자로만 작성
  • 이미지 3배수 처리한다. (원사이즈/3)
  • 폰트 : px, 폰트관련 width, height 도 px 을 사용한다
  • margin, padding, border, background 는 px로 사용한다
  • CSS의 주석은 시작만 작성
  • 객체 구분을 위한 코드 그룹간 빈 한 줄 허용(한 줄만 허용)
  • 컨텐츠 사이 간격을 위한 css("+" 선택자)는 파일내용 맨하단에 작성

4. 속성의 선언 순서

순서 속성 순서 속성
1 display 8 margin
2 visibility 9 padding
3 overflow 10 border
4 float 11 background(축약 속성 순서: img/ repeat/ 가로 정렬/ 세로 정렬)
5 position 12 color
6 width, max-width, min-width 13 font, letter-spacing, text-align, white-space
7 height, max-height, min-height 14 그 외 기타 속성

5. Class 명명 규칙

  • 시작의 이름은 영문 소문자를 사용하며 2개 이상의 단어 조합 경우 언더스코어(_)로 구분
  • 영문 소문자와 숫자만을 사용
  • 클래스명 숫자 증가시 언더스코어(_) 미사용
  • 단어 조합의 경우 3개를 초과하지 않는다.
  • 단어의 극단적 축약 및 긴 단어는 지양한다.
  • 각 폴더명으로 컨텐츠 영역 class 생성 ex) .ctg_폴더명(_확장가능)
